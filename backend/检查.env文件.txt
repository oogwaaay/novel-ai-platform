  如何检查和修复 .env 文件
═══════════════════════════════════════════════════════

【问题】错误：The OPENAI_API_KEY environment variable is missing

【解决步骤】

1. 检查文件是否存在
   - 打开 backend 文件夹
   - 查看是否有 .env 文件
   - 如果没有，需要创建

2. 检查文件位置
   - 文件必须在：novel-ai-platform\backend\.env
   - 不是在：novel-ai-platform\.env

3. 检查文件内容
   - 打开 .env 文件
   - 应该包含：
     PORT=3001
     DEEPSEEK_API_KEY=sk-你的密钥
     JWT_SECRET=abc123

4. 检查文件名
   - 必须是：.env（前面有点）
   - 不能是：.env.txt 或 env.txt

5. 验证文件内容
   - 在 backend 文件夹打开命令行
   - 输入：type .env
   - 应该能看到文件内容

6. 重启服务器
   - 修改 .env 后必须重启
   - 按 Ctrl+C 停止
   - 再运行：npm run dev
